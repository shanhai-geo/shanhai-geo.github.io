# 山海云枢 · Agent供电局架构 V2.0

## 核心理念

Agent调度 = Agent供电局
不是管理口的配件，而是独立的生态平台
跟GEO供电局同构：发电→输电→配电→用电

---

## Agent供电局四层飞轮

```
Layer 1: 用自己的Agent调度服务自己
  → 管理口主控室、内部决策、系统自运转
  → 零成本，纯内部消耗

Layer 2: 用自己的Agent调度服务自己的产品
  → GEO客户（298/998/4998）获得Agent能力
  → 白茶/酒类智能客服
  → 品牌矩阵自动服务
  → 实物产品+数字Agent捆绑

Layer 3: 卖Agent调度本身
  → 开放Agent调度API，按次/按月计费
  → 企业客户接入我们的Agent供电局
  → 白标方案：帮企业搭建私有Agent调度

Layer 4: 帮别人用Agent调度卖产品/做服务
  → 加盟伙伴获得自己的"mini Agent供电局"
  → 每个GEO节点=一个Agent供电局分电站
  → 生态内Agent互相服务，形成内部经济
```

## 安全模型: 三层隔离

```
Layer 0: 管理口内部 (零信任)
  ✗ 不含任何密钥
  ✗ 不直接调用外部API
  ✓ 本地指令零网络请求
  ✓ 系统数据纯本地计算

Layer 1: 穿透通道 (加密令牌)
  ✓ ADMIN_KEY鉴权
  ✓ HMAC签名防篡改
  ✓ 限流60次/分钟
  ✓ 审计日志

Layer 2: Agent供电局 (密钥集中)
  ✓ 所有密钥在环境变量
  ✓ 前端不可见
  ✓ 独立部署/升级
  ✓ 端口独立开关
```

## 管理口指令流

### 本地指令 (零网络)
```
!自检 → 本地执行
!健康度 → 本地数据
!芯片 → 本地芯片
!端口状态 → 调用网关查询各端口开关
```

### AI对话 (穿透网关)
```
用户输入问题
  → 管理口检测到非本地指令
  → 管理口用ADMIN_KEY调用/api/gateway
  → 网关路由到可用端口(P1-A/B/C/D)
  → AI服务返回
  → 网关返回管理口
  → 展示结果
```

### 主Agent深度模式
```
切换"主Agent模式"
  → 管理口调用/api/gateway?port=coze
  → 网关固定路由P1-D
  → Coze Bot返回完整Agent能力结果
```
