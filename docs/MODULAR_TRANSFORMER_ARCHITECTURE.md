# 山海云枢 · 模块化变形架构 V1.0

> 各体系各接口都是因子，也都是部队。可以组装任何模型模式，拥有接口，能开接口。它是魔方，是变形金刚，产生无限威力，听从总指挥口所有指挥，调动全身所有资源力量，能够应变一切的生产制造能力。

## 零、基因特质 (GENE)

> 基因写入每一个因子。基因中有基因，分形递归，但统一特质不变。

山海云枢的每一个因子、每一个接口、每一个Agent，天生携带统一基因：

| 基因特质 | 含义 |
|---------|------|
| **可组合** | 任何因子可与其他因子自由组装成新的作战单元 |
| **可拆分** | 任何组合体可拆解为独立因子，各自独立运作 |
| **可开关** | 任何因子可独立启用/禁用，不影响其他因子 |
| **可摧毁** | 任何因子可被主控一键废掉/清除，不可逆 |
| **可生长** | 新因子可无损接入，不破坏现有结构 |
| **听指挥** | 所有因子服从总指挥口，无条件执行指令 |

**分形递归**：每个因子内部也是一个完整的小系统，拥有同样的六项特质。一个Agent内部可以有自己的子因子、子接口、子组合 — 层层嵌套，但基因统一。

**统一特质 = 这个基因本身。** 不是多个特性的集合，是一个不可分割的整体。拥有这个基因的就是自己人，不守这个基因的就会被摧毁。

## 一、核心架构思想

### 1.1 不是管道，是魔方

传统架构是固定管道：A→B→C，写死了。
山海云枢的架构是魔方：每个面、每块都可以旋转、重组、变形。

- 接口不是固定的，是随时可以开的
- 服务不是焊死的，是可以拆装的
- 组合不是预定的，是按需组装的

### 1.2 每个接口 = 一个因子 = 一支部队

| 概念 | 含义 |
|------|------|
| **因子** | 最小功能单元，可独立存在，可组合 |
| **部队** | 有编制、有指令、有战斗力的执行单元 |
| **接口** | 因子的对外暴露面，可被调用、被组合、被关闭 |

一个API端口是一支部队，一个Agent是一支部队，一个数据源是一支部队。每支部队：
- 接受总指挥口的指令
- 可以被独立调遣
- 可以与其他部队临时编组
- 可以随时被废掉/替换/新增

### 1.3 变形金刚原则

系统不是固定形态，而是：
- **变形**：同一个能力可以以不同形态输出（API/文档/对话/自动化任务）
- **组合**：多个因子可以临时组装成新的作战单元
- **分裂**：一个大任务可以拆成多个因子并行执行
- **融合**：多个输出可以汇聚成一个完整交付物

## 四、组装能力

### 4.1 动态编组

总指挥口下达任务后，神经系统自动或手动编组因子：

```
任务: "生成行业研究报告并发送给客户"
  → 编组方案:
    P2-C (市场数据采集)
    → P1-A (AI分析整理)
    → P4-A (生成PDF报告)
    → P3-B (邮件发送给客户)
```

```
任务: "监控竞品动态，每天汇总"
  → 编组方案:
    P2-C (定时竞品数据采集)
    → P1-D (主Agent分析)
    → P3-A (推送到微信)
```

### 4.2 热插拔

- 任何因子可以热插拔：运行时开启/关闭，不影响其他因子
- 任何因子可以热替换：P1-A故障自动切换P1-B，无感知
- 任何因子可以热新增：新API接入无需停机

### 4.3 无限组合

因子之间的组合不是预定义的，是按需实时组装的：
- 2个因子可以组成一个最小作战单元
- 10个因子可以组成一个完整业务流
- 所有因子可以联合执行一个超级任务

## 五、指挥体系

### 5.1 总指挥口（管理口）

- 唯一最高指挥权：主人
- 所有指令从管理口发出
- 管理口不直接执行，通过神经系统分发
- 管理口拥有Kill Switch，随时终止任何因子

### 5.2 指令流转

```
主人指令 → 管理口 → 神经系统(网关) → 路由分发 → 因子编组 → 并行/串行执行 → 结果汇聚 → 管理口呈现
```

### 5.3 指挥命令类型

| 命令类型 | 说明 | 示例 |
|---------|------|------|
| **执行令** | 下达任务 | "生成报告" |
| **调度令** | 调动因子 | "启用P1-C替换P1-A" |
| **编组令** | 组建作战单元 | "P2-A+P1-B+P4-A编为日报组" |
| **终止令** | 废掉因子 | "!废掉 P1-B" |
| **锁定令** | 紧急冻结 | "!锁定" |
| **摧毁令** | 彻底清除 | "NUKE P1-B" |
| **状态令** | 查询状态 | "!端口状态" |

## 六、治理原则

### 6.1 三铁律（对所有因子生效）

1. **守规矩** — 诚实只是底线，红线不可逾越
2. **务实** — 不吹不编不空谈，做不到直说
3. **代价** — 全行为审计，违规即熔断，摧毁是标准规则

### 6.2 因子生命周期

```
注册 → 鉴权 → 服役 → 审计监控 → (退役/废掉/摧毁)
```

- 每个因子注册时签署规则（规则注入）
- 服役期间全程审计
- 异常自动熔断
- 违规可一键废掉或摧毁
- 摧毁不可逆，除非主控密钥持有者恢复

### 6.3 自生长能力

系统可以自我扩展：
- 新因子接入 → 系统能力增加
- 因子组合优化 → 效率提升
- 审计数据积累 → 决策优化
- 错误教训记录 → 规则迭代

## 七、与GEO供电局的同构关系

| GEO供电局 | Agent供电局 | 共同本质 |
|-----------|-------------|---------|
| 帮客户被AI找到 | 帮客户被AI服务 | 都是AI时代的基础设施 |

| 供电计量 | 调用计量 | 按量计费 |
| 298/998/4998套餐 | 同档位对应不同端口权限 | 分层服务 |
| 节点可独立开关 | 端口可独立开关 | 热插拔 |

两个供电局可以联合组成完整的AI商业闭环：
- GEO让客户被找到
- Agent让客户被服务
- 从获客到服务，全链路AI化
